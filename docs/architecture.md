# 專案架構與核心邏輯 (Architecture & Core Logic)

本專案是一個基於 **RIME (中州韻輸入法引擎)** 的台語/客語等語言輸入法方案，涵蓋了白話字 (TLPA)、方音符號 (Bopomofo) 以及十五音等各種輸入規則與詞庫。

## 📁 專案目錄結構分析

專案的架構主要分為「輸入法核心設定」、「動態處理腳本」、「開發與部署工具」以及「文件庫」四大區塊。

### 1. 核心輸入法配置與詞庫 (專案根目錄 `/`)
專案的靈魂所在，由宣告式的 YAML 檔案構成，定義了所有的輸入規則與字詞對應。

*   **`*.schema.yaml`**: 各種輸入法方案的定義檔（例如：`tlpa_hong_im.schema.yaml`, `bp_hong_im.schema.yaml`），決定了鍵盤對應、拼寫規則與轉換行為。
*   **`*.dict.yaml`**: 詞庫檔案（例如：`tl_ji_khoo.dict.yaml`），包含了大量的漢字與拼音對應資料。
*   **`lib_*.yaml` / `*_libs.yaml`**: 模組化的共用設定檔，被各個 schema 引入，減少重複配置。
*   **`default.custom.yaml` / `weasel.custom.yaml`**: 定義 RIME 引擎與 Windows 用戶端 (小狼毫) 的全域客製化行為。

### 2. 進階動態處理邏輯 (`/lua/` 與 `/rime.lua`)
當 YAML 設定無法滿足複雜的輸入需求時，專案透過 Lua 腳本來擴充 RIME 的功能。

*   **`/lua/`**: 存放各種自訂的 Translator 與 Filter 腳本，用於動態轉換拼音、過濾或重新排序候選詞。
*   **`rime.lua`**: Lua 腳本的載入入口，RIME 引擎會透過此檔案註冊所有的 Lua 擴充功能。

### 3. 開發、建置與部署工具 (`/src/`, `/tools/`, 及根目錄腳本)
負責專案的自動化建置、環境檢查與安裝程式打包。

*   **`/src/`** 與 **`/tools/`**: 包含輔助開發、詞庫轉換、資料驗證的 Python 程式碼或工具腳本。
*   **根目錄的 `*.ps1`, `*.bat`, `*.py`**: 用於檢查 Windows EUDC (造字區) 環境、自動化部署 RIME 設定，以及開發過程中的測試腳本。
*   **`*.spec`**: PyInstaller 的設定檔，用於打包驗證工具或安裝程式 (Installer)。

### 4. 產出物與發布 (`/build/`, `/release/`)
*   存放編譯後的執行檔、安裝程式以及準備發布的版本檔案。

### 5. 文件與規格 (`/docs/`)
*   包含版本發行指引、安裝 SOP、輸入法規格書（如《河洛白話輸入法規格.md》）、教學投影片與設計試算表。

## 🎯 開發指南：我該去哪裡修改程式？

*   **修改打字規則、鍵盤對應或擴充詞庫** 👉 請至 **專案根目錄** 修改對應的 `.schema.yaml` 或 `.dict.yaml`。
*   **修改進階的動態選詞行為、特殊拼音轉換或過濾機制** 👉 請至 **`lua/`** 目錄修改 Lua 腳本，並確認 `rime.lua` 有正確載入。
*   **修改安裝程式邏輯、開發工具或建置流程** 👉 請至 **`src/`**、**`tools/`** 或檢視根目錄的 PowerShell/Python 腳本。
