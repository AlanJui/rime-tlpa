# 閩南話漢字標音轉換指引

`版本：V0.1`

本文件用於說明，如何以【台語音標】（TLPA+）作為**漢字讀音互換使用之核心標準**。

【註】：

1. 為求本文件撰寫之簡潔、易讀，TLPA+以下將簡寫成：**TLPA**。

2. **TLPA+** 為 TLPA 之**改良版**，兩者之間的差異在於：TLPA+
   的【羅馬拼音字母】均為單一字元；且其羅馬拼音字母之使用與【漢語拼音】有著更高之相容度：

| 台羅拼音 | TLPA | TLPA+ |
| :------: | ---- | ----- |
|   tsh    | ch   | c     |
|    ts    | c    | z     |

---

## 1. 交換核心標準：台語音標

## 2. 各標音系統轉換執行重點

### A. 台羅拼音 (Tâi-lô)

#### 聲母轉換

- `z` → `ts`
- `c` → `tsh`

#### 韻母轉換

- onn --> oonn

  **韻母變更**：保持 TLPA 原始結構（如 `ue`, `ua`, `ik`, `ing` 均不變）。

#### 聲調轉換

【台語音標】使用【調號】；但【台羅拼音】、【白話字】使用【調符】（聲調符號）。

| 1        | 陰平         | 44   | 1     | 高音調 | 東   | tong1    | tong         |              | tong     |

**調符位置**：遵循優先級規則 (a > o > e > i > u)。

- 標記之調號為該對應漢字之本調，至於實際讀出的聲調則需以連續變調之規則進行變調。外來語則擬似本調。
- 當一個音節有多個字母時，調號得標示在響度最大的字母上面（通常在韻腹）。響度順序： a > oo > (e = o) > (i = u)
- 雙母音 iu 及 ui ，調號都標在後一個字母上，因為前一個字母是介音。
- m 作韻腹時標於字母 m 上。 oo 及 ng，標於前一個字母上；比如 ng 標示在字母 n 上。
- 三合字母 ere，標於最後的字母 e 上。

### B. 白話字 (Pe̍h-ōe-jī, POJ)

#### 聲母轉換

- z → ch
- c → chh

#### 韻母轉換

以下為【韻母】在 TLPA 轉換成 POJ 的規則：

- ue → oe
- ua → oa
- ik → ek
- ing → eng
- oo → o\u0358 (o 右上方加點：o͘)
- nn → \u207F (韻母鼻音化識別符格式：ⁿ)
- nnh → h\u207F (入聲韻母鼻音化格式：hⁿ)

#### 聲調轉換

參考【台羅拼音】作法。

### C. 方音符號 (TPS)

#### 聲母轉換

- 雙唇音（清聲送氣塞音/清聲不送氣塞音/濁聲不送氣塞音/濁聲鼻音）：
  - ph → ㄆ [ph]
  - p → ㄅ [p]
  - b → ㆠ [b]
  - m → ㄇ [m] (鼻音)

##### 零聲母轉換

###### 【零聲母】+【韻母開頭為 i】

以 `i` 開頭：若後接韻母則 `i` 變 `y`，若僅有 `i` 則加 `y` 前綴。

1. 當【韻母】以【i】開頭，且【i】為【主要元音】，則在【i】之前加【y】，如：
   - 伊： i --> yi
   - 因： in --> yin
   - 音： im --> yim
   - 英： ing --> ying
   - 一： it --> yit
   - 揖： ip --> yip
   - 益： ik --> yik

2. 當【韻母】以【i】開頭，且【i】當作【介音】之韻，則【i】變更成【y】，如：
   - 枵： iao --> yao
   - 煙： ian --> yan
   - 鞅： iang --> yang
   - 擁： iong --> yong

3. 以【i】開頭的【鼻化韻母】（如：ni, nih, nia, niah, nioo, niu, nao, niaoh），則寫成：
   - 圓： ni --> yna
   - 營： nia --> yna
   - 喓： niao --> ynao
   - 羊： niu --> ynu
   - 盈： nih --> ynih

###### 【零聲母】+【韻母開頭為 u】

以 `u` 開頭：若後接韻母則 `u` 變 `w`，若僅有 `u` 則加 `w` 前綴。

1. 【u】為【主要元音】，則在【u】之前加【w】，如：
   - 有： u --> wu
   - 溫： un --> wun
   - 兀： ut --> wut

2. 【u】當作【介音】之韻，則【u】變更成【w】，如：
   - 彎： uan --> wan
   - 歪： uai --> wai
   - 位： ui --> wi
   - 衛： ue --> we

3. 【u】開頭的【鼻化韻母】（如：nu, nua, nue, nueh, nui, nuai, nuaih），則寫成：
   - 碗： nua --> wna
   - 彎： nuai --> wnai

## 3. 調符套用邏輯 (Tone Mark Logic)

### 優先級規則 (Placement Priority)

當一個音節有多個母音時，調符應標註於權重最高的字母上：

1.  **a** (最高)
2.  **oo / o**
3.  **e**
4.  **i / u** (後者優先，如 `ui` 標在 `i`，`iu` 標在 `u`)

### 組合字元 (Combining Diacritics)

- **第 8 聲 (陽入調)**：必須使用 `\u030D` (Combining Vertical Line Above)。
- **標準化**：輸出前應執行 `normalize("NFD")`，這能讓字型（如 Charis SIL）更好地處理標記定位。

## 4. 關鍵技術修復與美學規範

以下是本系統開發過程中發現的關鍵技術要領，對標音穩定度至關重要：

### I. 零聲母對齊修復 (The ZWSP Hack)

- **問題**：瀏覽器在渲染以母音開頭（零聲母）且帶有組合調符（如 `e̍k`）的字串時，常會發生對齊位移。
- **解決方案**：在所有標音字串前端注入 **`\u200B` (零寬度空格)**。這能為調符提供一個穩定的錨點，確保對齊完美。

### II. 靜態 HTML 結構緊湊化

- **規範**：生成的 HTML 源碼中，`<ruby>`、`<rt>`、`<rtc>` 標籤之間**嚴禁換行或空格**。
- **理由**：HTML 內的任何空白字元在瀏覽器中都會被轉換為可視空格，導致漢字間距被拉開，特別是在手機 RWD 模式下會造成排版崩潰。

### III. 響應式字體縮放 (RWD Title)

- **規範**：在小螢幕（< 600px）下，標題字體應自動降級至 `1.3rem ~ 1.5rem`，並強制 `white-space: nowrap`。
- **理由**：標題（如《寒窯賦》）若因螢幕太窄而折行，會嚴重影響導讀體驗。

### IV. 字型依賴 (Font Sovereignty)

- **規範**：系統必須內建並載入 `Charis SIL` 字型。
- **功能需求**：CSS 必須啟用 `font-feature-settings: "ccmp" 1, "mark" 1, "mkmk" 1;`。
- **理由**：這是目前唯一能保證台羅/白話字調符不位移、不重疊的工業級解決方案。
