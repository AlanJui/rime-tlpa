# 閩拼方案（BP）候選清單實作指引

> 適用檔案：`lib_comment_bp.yaml`  
> 版本：v0.1.0.12 以後

## 1. 零聲母 + i/u 開頭韻母 — 為何可以正常轉換 y/w？

### 問題背景

閩拼方案規定，零聲母音節若韻母以 `i` 或 `u` 開頭，在書寫時必須依規則加上前導字母：

| 字典原始音標 | 書寫顯示 | 規則 |
|---|---|---|
| i1（衣） | yī | 前加 y |
| it4（一） | yīt | 前加 y |
| uan2（遠） | wǎn | u 改 w |
| u1（於） | wū | 前加 w |

### 整個處理流程

```
字典值（如 uan2）
  │
  ▼
【第一步】BPM2→BP 聲調轉換：xlit|234567|357246|
  uan2 → uan3
  │
  ▼
【第二步】音節標記包裝（line 46-48）
  uan3 → 〔〉Øuan3〕 【》Øuan3】
  │
  ▼
【第三步】移除零聲母符號 Ø
  〔〉Øuan3〕 → 〔〉uan3〕
  │
  ▼
【第四步】i/u 零聲母校調（lib_comment_bp.yaml line 258-284）
  〔〉uan3〕 → 〔wan3〕    ← u(a[inp]?) 命中
  │
  ▼
【第五步】聲調符號上調（xform/a3/ǎ/）
  〔wan3〕 → 〔wǎn〕
```

### 關鍵一：`〕` 做音節邊界，而非行尾 `$`

`comment_format` 的規則套用在整個候選字的 comment 字串上，其結構為：

```
〔〉uan3〕 【》uan3】
```

字串並不在 `uan3` 後結尾，而是後面還有空白與右欄的 `【...】`。  
因此若用 `$` 做邊界：

```yaml
- xform/〔〉u(a[inp]?)(\d)$/〔w$1$2/   # ✗ 永遠不會命中
```

因為 `$` 要求字串到此結束，但實際上後面還有 ` 【》uan3】`，所以永遠失敗。

改用 `〕` 做音節結尾邊界後才能正確命中：

```yaml
- xform/〔〉u(a[inp]?)(\d)〕/〔w$1$2〕/  # ✓ 命中 〔〉uan3〕
```

### 關鍵二：ia/io/iu 用前綴替換，不加行尾限制

`ia/io/iu` 後面還有更多字母（如 `iao、iang、iong`），無法用終止型比對，  
必須用「前綴直接替換」：

```yaml
- xform/〔〉ia/〔ya/   # 〔〉iauX〕 → 〔yauX〕，後面繼續讓聲調規則處理
- xform/〔〉io/〔yo/
- xform/〔〉iu/〔yu/
```

但 `i` 後面若是直接接調號（如 `i1`）或輔音（`in/it/im/ip/ing/ik`），  
後面**一定**接 `〕`，就可以安全用 `〕` 做邊界：

```yaml
- xform/〔〉i(\d)〕/〔yi$1〕/          # i1 → yi1
- xform/〔〉i(n|t)(\d)〕/〔yi$1$2〕/   # it4 → yit4
```

### 規則順序必須「複雜樣式先於簡單樣式」

RIME 的 xform 是逐條套用、一旦命中就不再繼續比對同一字串段落（但規則本身仍依序套用）。  
因此複合韻母一定要先於單韻母：

```yaml
- xform/〔〉i(ang|ak)(\d)〕/〔y$1$2〕/   # 先：iang/iak
- xform/〔〉i(ong|ok)(\d)〕/〔y$1$2〕/   # 先：iong/iok
- xform/〔〉i(a[inp]?)(\d)〕/〔y$1$2〕/  # 先：ian/iam/iao/iap
- xform/〔〉i(a)(\d)〕/〔y$1$2〕/        # 後：ia
- xform/〔〉i(\d)〕/〔yi$1〕/            # 最後：i（純元音）
```

若反過來把 `i(\d)` 擺最前面，`it4` 會被 `i(\d)` 誤命中（i 後面是 t，不是數字，所以其實不會命中，但仍要注意順序邏輯）。  
`ia/io/iu` 的前綴規則一定要排在所有終止型規則**之前**，否則 `〔〉io` 還沒被替換，後面的 `iong/iok` 終止型就已先搶走。

## 2. 漢字連續輸入 — 為何加入 line 46-48 就解決？

### 問題背景

輸入「黃河遠一」時，輸入串為 `hong5 ho5 uan2 it4`（或連字 `hong5ho5uan2it4`）。  
RIME 的 comment_format 接到的是**整個多音節字串**，例如：

```
hong5 ho5 uan2 it4
```

若用行首 `^` 錨點，只能命中第一個音節；其後的音節不會被任何規則碰到，  
導致後三個音節出現在候選字的 comment 欄時完全未被轉換。

### 解法：用 `\b` 詞界逐個音節包裝

Line 46-48：

```yaml
- xform/\b((?:m|ng)\d)\b/〔〉Ø$1〕 【》Ø$1】/
- xform/\b([iu]?(?:ai|ao|oo|[aiueo])[a-z]*\d)\b/〔〉Ø$1〕 【》Ø$1】/
- xform/\b([a-z]+\d)\b/〔〉$1〕 【》$1】/
```

`\b` 是詞界（word boundary），在空白隔開的每個 token 兩端都有詞界，  
所以三條規則會**逐個命中每一個音節 token**，不論字串裡有幾個音節：

```
hong5 ho5 uan2 it4
  ↓ 三條規則各自命中
〔〉hong5〕 【》hong5】 〔〉Øho5〕 【》Øho5】 〔〉Øuan2〕 【》Øuan2】 〔〉Øit4〕 【》Øit4】
```

### 三條規則的分工

| 行 | 目標 | 說明 |
|---|---|---|
| 46 | `m2`、`ng5` 等韻化輔音音節 | 鼻音單音節必須先攔截，否則被行 48 的通用規則包成非零聲母 |
| 47 | `uan2`、`i1`、`ao3`… 等零聲母音節 | `[iu]?(?:ai|ao|oo|[aiueo])[a-z]*\d`：無聲母起頭，加上零聲母標記 Ø |
| 48 | `hong5`、`kit4`… 等有聲母音節 | 通用兜底，所有剩餘有聲母的音節 |

### 為何行 47 要包含 `[iu]?`

台語音節中，複合韻母如 `iau/ian/uai/ui` 等也是零聲母，不帶聲母字母。  
正則 `[iu]?(?:ai|ao|oo|[aiueo])[a-z]*\d` 的拆解：

```
[iu]?          → 可選的 i 或 u（介音）
(?:ai|ao|oo|[aiueo])  → 必有一個核心元音
[a-z]*         → 後面可接任意輔音韻尾（如 ng/k/n/t/m/p 等）
\d             → 調號數字
```

這樣才能命中 `iau1`、`uai3`、`ian2`，而不只是 `a1`、`u5`。

### 如果不用 `\b`，改用 `^` 會怎樣？

```yaml
# ✗ 這樣只能命中第一個音節
- xform/^([iu]?...)\d/$1/
```

多音節輸入時，第二個音節開始都不在字串開頭，`^` 不命中，  
整個後段的轉換規則（聲母改寫、聲調符號、y/w 替換）全部失效，  
候選字的 comment 欄就會出現還沒轉換的原始數字調 BPM2 音標。

## 3. 實作時的必記原則

### 3.1 邊界符號選用

| 情境 | 用什麼邊界 | 原因 |
|---|---|---|
| comment_format 單音節字串 | `$`（行尾）或 `〕`（結尾識別符） | 確保只命中當前音節 |
| comment_format 多音節字串 | 先用 `\b` 包裝每個音節，再對 `〔...〕` 內部操作 | 不用 `\b` 後段音節無法被命中 |
| preedit_format | `^`（行首）配合每條規則 | preedit 每次只處理一個輸入串（單音節或連字） |
| speller/algebra | `^...$` 完整音節 | algebra 每條規則對單一音節操作 |

### 3.2 規則書寫順序

```
複合韻母 > 複合輔音韻尾 > 單韻母 > 純元音
iang/iong/uang > ian/iam/uat > ia/ua > i/u
```

### 3.3 左右欄識別符號

本方案採用左右雙欄格式：

```
〔〉{音標}〕  ← 左欄：閩拼（BP）顯示
【》{音標}】  ← 右欄：方音符號（台語注音）顯示
```

- 左欄對左欄操作時，一律對 `〔〉...〕` 內部做 xform
- 右欄對右欄操作時，一律對 `【》...】` 內部做 xform
- **不可混用**，否則會造成欄位污染或跨欄意外匹配

### 3.4 零聲母標記 Ø 的生命週期

```
①包裝時插入 Ø   →  ②各欄分別消除 Ø   →  ③再做各欄轉換
〔〉Øi1〕           〔〉i1〕                 〔yi1〕
【》Øi1】           【》i1】                 【ㄧ》1】
```

務必在**消除 Ø 之後、開始字母轉換之前**，確認識別符號 `〉` 仍在，  
這樣後續規則才能精準用 `〔〉` 或 `【》` 做前綴比對。
